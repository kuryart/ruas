# Internacionalização (i18n)

O Ruas usa o sistema **Mozilla Fluent** para internacionalização. As traduções ficam em arquivos `.ftl` organizados por idioma e módulo.

**Arquivos principais:**
- `frontend/src/i18n/context.tsx` — bundles, provider e hook
- `frontend/src/i18n/detect.ts` — detecção do idioma do sistema
- `frontend/src/locales/<locale>/<module>.ftl` — arquivos de tradução

---

## Layout dos arquivos de tradução

```
frontend/src/locales/
├── en-US/
│   ├── common.ftl       ← strings genéricas (save, cancel, loading, módulos)
│   ├── vault.ftl        ← tela de seleção/criação de vault
│   ├── contacts.ftl     ← módulo de contatos
│   ├── workspace.ftl    ← workspace, painéis, splits
│   ├── settings.ftl     ← modal de settings
│   └── notes.ftl        ← módulo de notas
└── pt-BR/
    ├── common.ftl
    ├── vault.ftl
    ├── contacts.ftl
    ├── workspace.ftl
    ├── settings.ftl
    └── notes.ftl
```

O idioma padrão é `pt-BR`. O fallback quando uma chave está ausente é o ID da chave em si.

## API do sistema i18n

### Tipos

```typescript
type Locale = 'pt-BR' | 'en-US'
type FluentArgs = Record<string, FluentVariable>
type TranslateFn = (id: string, args?: FluentArgs) => string
```

### `I18nProvider`

```tsx
// Envolve o app inteiro (App.tsx)
<I18nProvider locale={detectLocale()}>
  {props.children}
</I18nProvider>
```

### `useI18n()`

```typescript
const { t, locale, setLocale } = useI18n()

// Tradução simples
t('contacts-header')          // → "Contatos" (pt-BR)

// Tradução com variáveis
t('notes-status-saving', { count: 3 })

// Mudança de idioma
setLocale('en-US')
```

Em desenvolvimento, chaves ausentes emitem `console.warn`.

## `buildBundle(locale, ftlStrings)`

```typescript
function buildBundle(locale: Locale, ftlStrings: string[]): FluentBundle
```

Cria um `FluentBundle` a partir de múltiplos arquivos `.ftl` importados via `?raw`. A opção `useIsolating: false` desabilita os caracteres bidi de isolamento (necessário para UI compacta).

Os bundles são construídos uma vez ao inicializar o módulo e guardados no record `BUNDLES`.

## `detectLocale()`

```typescript
// Constantes em detect.ts:
const SUPPORTED: Locale[] = ['pt-BR', 'en-US']
const DEFAULT: Locale = 'pt-BR'

function detectLocale(): Locale
  // 1. Verifica navigator.languages (array em ordem de preferência)
  // 2. Para cada lang: tenta match exato ('pt-BR'), depois match por base ('pt')
  // 3. Retorna DEFAULT se nenhum match encontrado
```

## Sintaxe Fluent básica

```fluent
# Mensagem simples
save = Salvar

# Com variável
contacts-count = { $count } contatos

# Plural
files-count = { $count ->
    [one] 1 arquivo
   *[other] { $count } arquivos
}

# Atributo (para tooltips, placeholders, etc.)
search-input =
    .placeholder = Buscar...
    .aria-label = Campo de busca
```

Para a referência completa da sintaxe Fluent: https://projectfluent.org/fluent/guide/

## Convenção de nomenclatura de chaves

O padrão usado nos arquivos existentes é `módulo-entidade-ação`:

| Exemplos | |
|---|---|
| `contacts-header` | Título do painel de contatos |
| `contacts-new-btn-title` | Tooltip do botão "novo contato" |
| `contact-add-email` | Botão "adicionar e-mail" em um contato |
| `notes-status-saved` | Status "salvo" no rodapé da nota |
| `notes-fm-add-tag` | Botão "adicionar tag" no frontmatter |
| `settings-category-general` | Categoria "Geral" nas settings |

## Checklist para adicionar um novo arquivo de locale

1. Crie `frontend/src/locales/en-US/<módulo>.ftl` com todas as chaves em inglês
2. Crie `frontend/src/locales/pt-BR/<módulo>.ftl` com as traduções em português
3. Em `context.tsx`, importe ambos os arquivos via `?raw`:
   ```typescript
   import enMeuModulo from '../locales/en-US/meu-modulo.ftl?raw'
   import ptMeuModulo from '../locales/pt-BR/meu-modulo.ftl?raw'
   ```
4. Adicione ao array de cada locale no objeto `BUNDLES`:
   ```typescript
   'en-US': buildBundle('en-US', [...existentes, enMeuModulo]),
   'pt-BR': buildBundle('pt-BR', [...existentes, ptMeuModulo]),
   ```

## Chaves mínimas por módulo

Todo novo módulo de UI deve incluir pelo menos estas chaves no `.ftl`:

| Chave padrão | Uso |
|---|---|
| `<módulo>-header` | Título do painel/lista |
| `<módulo>-new-btn-title` | Tooltip do botão de criação |
| `<módulo>-search-placeholder` | Placeholder do campo de busca |
| `<módulo>-loading` | Texto de carregamento |
| `<módulo>-empty` | Texto quando a lista está vazia |
| `<módulo>-no-results` | Texto quando busca não retorna resultados |
| `<entity>-status-saved` | Status "salvo" |
| `<entity>-status-unsaved` | Status "não salvo" |
| `<entity>-status-saving` | Status "salvando" |
| `<entity>-status-error` | Status de erro ao salvar |
